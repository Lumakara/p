# Design Decision Framework

## Urutan keputusan
1. Pahami tujuan halaman.
2. Cek PRD dan philosophy.
3. Cek referensi visual.
4. Terapkan color system dan typography.
5. Susun layout berdasarkan hierarchy.
6. Tambahkan micro-interactions bila ada alasan.
7. Cek responsive behavior.
8. Cek accessibility dan performance.
9. Review ulang untuk konsistensi.

## Saat ada konflik
- UX menang atas dekorasi.
- Dokumen proyek menang atas referensi.
- Simplicity menang atas kompleksitas yang tidak perlu.
- Performance menang atas efek yang tidak penting.

## Definisi "bagus"
Bagus berarti:
- cepat dipahami,
- nyaman dipakai,
- terasa premium,
- konsisten,
- dan tidak terasa dibuat asal.
