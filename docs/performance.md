# Performance

## Tujuan
Frontend harus terasa cepat, ringan, dan nyaman di semua device.

## Prinsip performa
- Prioritaskan render awal yang cepat.
- Gunakan komponen yang efisien.
- Hindari library berat bila tidak perlu.
- Optimalkan image.
- Hindari efek visual berat yang tidak memberi manfaat.

## Asset strategy
- Gunakan format modern jika memungkinkan.
- Logo dan icon sebaiknya SVG.
- Gambar produk harus dioptimalkan.
- Gunakan lazy loading untuk konten yang tidak langsung terlihat.

## Motion performance
- Gunakan animasi yang GPU-friendly bila memungkinkan.
- Hindari animasi layout berat secara berlebihan.

## Code performance
- Komponen harus modular tetapi tidak over-engineered.
- Jangan duplikasi logic.
- Jangan membawa dependency tanpa alasan.
- Gunakan data fetching yang efisien.

## Target rasa
- Interaksi cepat
- Scroll mulus
- UI terasa ringan
- Tidak ada jeda yang tidak perlu
